# 데이타베이스 추가계획 컬럼

작성일: 2026-04-19  
목적: 디지털 상품(이메일 링크 발송형) 운영을 위한 1차 필수 컬럼 정리

## 전제
- 상품은 웹 디지털 상품이다.
- 배송지 정보는 사용하지 않는다.
- 결제 추적과 링크 발송/다운로드 추적을 우선한다.

## 1) `orders` 추가 컬럼

| 컬럼명 | 타입(권장) | 설명 |
|---|---|---|
| `order_no` | text unique | 사람이 읽는 주문번호 |
| `updated_at` | timestamptz | 마지막 수정 시각 |
| `paid_at` | timestamptz | 주문 결제 완료 시각 |
| `cancelled_at` | timestamptz | 주문 취소 시각 |

## 2) `payments` 추가 컬럼

| 컬럼명 | 타입(권장) | 설명 |
|---|---|---|
| `payment_method` | text | 결제수단(카드, 계좌이체 등) |
| `pg_transaction_id` | text unique | 결제사 거래번호(장애 추적 핵심) |
| `approved_at` | timestamptz | 결제 승인 시각 |
| `failed_at` | timestamptz | 결제 실패 시각 |
| `failure_code` | text | 결제 실패 코드 |
| `failure_message` | text | 결제 실패 설명 |

## 3) 신규 테이블 `digital_deliveries` (권장)

역할: 이메일 발송, 링크 클릭, 실제 다운로드 시작을 분리 기록

| 컬럼명 | 타입(권장) | 설명 |
|---|---|---|
| `id` | uuid pk | 고유 ID |
| `order_id` | uuid fk | 주문 연결 |
| `order_item_id` | uuid fk | 주문 상품 연결(상품별 추적용) |
| `delivery_email` | text | 발송 대상 이메일 |
| `access_token` | text unique | 다운로드 접근 토큰 |
| `token_expires_at` | timestamptz | 토큰 만료 시각 |
| `sent_at` | timestamptz | 이메일 발송 시각 |
| `first_clicked_at` | timestamptz | 링크 첫 클릭 시각 |
| `download_started_at` | timestamptz | 다운로드 버튼 클릭 시각 |
| `download_count` | integer default 0 | 다운로드 시작 횟수 |
| `last_downloaded_at` | timestamptz | 마지막 다운로드 시작 시각 |

## 운영 메모
- `클릭`과 `다운로드 시작`은 별도 이벤트로 본다.
- 2~4MB 파일 기준에서는 대용량 전송 최적화보다 추적 로그 분리가 더 중요하다.
