- 사용자가 account 페이지에서 답변 확인

### 4-1. DB 테이블 추가 (Supabase)
```sql
-- inquiries: 상담/문의 통합 테이블
- id (uuid)
- profile_id (uuid FK)
- type ('consulting' | 'general')
- title, content
- status ('pending' | 'in_progress' | 'resolved' | 'closed')
- has_unread_reply (boolean) - 관리자 답변 여부
- created_at, updated_at

-- inquiry_replies: 답변 이력
- id (uuid)
- inquiry_id (uuid FK)
- author_role ('customer' | 'admin')
- content, created_at
```

**파일:** `supabase/schema.sql` 수정 (테이블 + 인덱스 추가)

**주요 함수:**

### 4-3. API 라우트 작성 (5개)

#### 사용자 API
1. `app/api/account/notifications/route.ts` (GET)
   - 미읽은 알림 카운트 반환
   - 헤더 배지가 60초마다 이 API를 호출

2. `app/api/account/consulting/route.ts`
   - GET: 문의 목록 조회 (타입, 페이지 필터)
   - POST: 새 문의 작성

3. `app/api/account/consulting/[id]/route.ts`
   - GET: 상세 조회 + 답변 목록 + 자동 읽음처리
   - PATCH: 문의 종료 (action: 'close')

#### 관리자 API
4. `app/api/admin/consulting/route.ts` (GET)
   - 전체 문의 목록 (사용자 이메일 JOIN)
   - 타입, 상태 필터링

5. `app/api/admin/consulting/[id]/route.ts`
   - GET: 상세 조회 + 답변 이력
   - POST: 답변 작성 (reply 생성 + 플래그 업데이트)
   - PATCH: 상태 변경

**인증:** 모두 Clerk `auth()` + `syncProfile()` 패턴 사용
